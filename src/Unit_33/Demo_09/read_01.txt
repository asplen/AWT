*******************************************************************************
JList
_______________________________________

Базовый класс списков в Swing называется JList.
Он поддерживает выбор одного или нескольких элементов из списка.

Хотя список часто состоит из строк, можно создавать списки практически
из любых объектов, допускающих отображение.

Класс JList настолько широко применяется в Java, что почти наверняка
вы встречались с ним неоднократно.

В прошлом элементы в JList представлялись как ссылки Object.
Однако, начиная с версии JDK 7, класс JList стал обобщенным и теперь он объявлен
следующим образом:

class JList<E>

Здесь Е представляет тип элементов в списке.
В JList определено несколько конструкторов, один их которых будет использоваться в главе:
JList (E [] items)

Он создает объект JList, содержащий элементы из массива items.

Класс JList основан на двух моделях.

Первая из них - интерфейс ListModel, который определяет, как осуществляется доступ к
данным списка.
Вторая модель - интерфейс ListSelectionModel, предлагающий методы, которые определяют,
какой элемент или элементы списка выбраны.

Несмотря на то что компонент JList работает надлежащим образом и сам
по себе, большую часть времени он помещается внутрь JScrollPane.

В результате длинные списки будут автоматически прокручиваться, что упрощает
проектирование графических пользовательских интерфейсов и также облегчает изменение
количества записей в списке без изменения размера компонента JList.

Компонент JList генерирует событие ListSelectionEvent,
когда пользователь делает выбор либо изменяет его.

То же самое событие генерируется при отмене пользователем выбора элемента.
Оно обрабатывается реализацией ListSelectionListener. В этом прослушивателе имеется
только один метод по имени valueChanged ():

void valueChanged (ListSelectionEvent le)

В le передается ссылка на событие.

Хотя ListSelectionEvent предоставляет ряд собственных методов, обычно для выяснения, что произошло, будет
опрашиваться сам объект JList.

Класс ListSelectionEvent и интерфейс ListSelectionListener находятся в пакете javax.swing.event.

По умолчанию компонент JList позволяет выбирать несколько диапазонов элементов в списке, но такое
поведение можно изменить, вызвав метод setSelectionMode (), который определен в JList:
void setSelectionMode (int mode)

В mode указывается режим выбора, который должен быть одним из следующих значений,
определенных в ListSelect ionModel:
SINGLE_SELECTION                    позволяет выбирать только одиночный элемент.
SINGLE_INTERVAL_SELECTION           выбирать один интервал элементов.
MULTIPLE_INTERVAL_SELECTION         несколько интервалов элементов в списке. (По умолчанию)
    Разумеется, одиночный элемент можно выбирать и в двух других режимах.
    Просто они также допускают выбор диапазона.

Чтобы получить индекс первого выбранного элемента, который также будет индексом единственного
выбранного элемента в режиме SINGLE_SELECTION, понадобится вызвать метод getSelectedindex ():
int getSelectedIndex ()
Индексация начинается с нуля. Таким образом, если выбран первый элемент,
то getSelectedIndex () возвратит О, а если элемент не выбран, то -1.

Вместо индекса выбранного элемента можно получить ассоциированное с
ним значение, вызвав метод getSelectedValue ():
Е getSelectedValue ()

Он возвращает ссылку на первое выбранное значение или null, если ни
одно значение не выбиралось.

В следующей программе демонстрируется применение простого компонента JList,
содержащего список городов.
Каждый раз, когда в списке выбирается какой-нибудь город, генерируется событие ListSelectionEvent,
которое обрабатывается методом valueChanged (), определенным в
ListSelect ionListener.
Он получает индекс выбранного элемента и отображает название выбранного города внутри метки.
