*******************************************************************************
Источники событий
_________________________________________________________________________
Источник представляет собой объект, который генерирует событие.

Это происходит, когда внутреннее состояние данного объекта каким-либо образом изменяется.

Источники могут генерировать более одного типа событий.
Источник должен зарегистрировать прослушиватели, чтобы они могли
получать уведомления о событиях конкретного типа. Для каждого типа события
предусмотрен свой метод регистрации. Вот общая форма:

public void addTypeListener ( TypeListener el )

Туре      указывается имя события,
el        ссылка на прослушиватель событий.

Например, метод, который регистрирует прослушиватель событий
клавиатуры, называется addKeyListener (), а метод,
регистрирующий прослушиватель движения мыши,
имеет имя addМouseMotionLis tener ().
Когда происходит событие, все зарегистрированные прослушиватели уведомляются
и получают копию объекта события, что называется

групповой рассылкой события.

Во всех случаях уведомления отправляются только тем прослушивателям,
которые зарегистрировались для их получения.

Некоторые источники могут допускать регистрацию только одного прослушивателя.

public void addTypeListener ( TypeListener el) throws java.util.TooManyListenersException


Туре    передается имя события,
el      ссылка на прослушиватель событий.

Когда такое событие происходит, зарегистрированный прослушиватель
уведомляется, что называется

индивидуальной рассылкой события.

Источник также обязан предоставлять метод, который позволяет
прослушивателю отменять регистрацию для события конкретного типа и имеет
следующую общую форму:

public void removeTypeListener ( TypeListener el )

Туре    указывается имя события,
el      ссылка на прослушиватель событий.

Скажем, чтобы удалить прослушиватель событий клавиатуры,
потребуется вызвать метод removeKeyListener () .

Методы для добавления и удаления прослушивателей предоставляются
источником, генерирующим события.
Например, Component, который является классом верхнего уровня,
определенным в АWT, предлагает методы добавления и удаления
прослушивателей событий клавиатуры и мыши.
