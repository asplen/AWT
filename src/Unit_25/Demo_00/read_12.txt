*******************************************************************************
Класс KeyEvent
_________________________________________________________________________

обытие типа KeyEvent генерируется, когда происходит ввод с клавиатуры.

Существуют три типа ключевых событий, которые идентифицируются
целочисленными константами

КЕУ_PRESSED     КЕУ_RELEASED        КЕУ_ТУРЕ

События первых двух типов возникают при нажатии или отпускании любой
клавиши, а события третьего типа - только в случае появления символа.
Помните, что не все нажатия клавиш дают в результате символ. Скажем,
нажатие <Shift> не производит символ.


В классе KeyEvent определено множество других целочисленных констант.
Например, константы VK_0-VK_9 (задают коды виртуальных клавиш и не зависят
от модифицирующих клавиш, подобных <Control>, <Shift> или <Alt>),
VK_A-VK_Z определяют АSСII эквиваленты цифр и букв.
Ниже перечислены остальные константы:
VK_ALT
VK_CANCEL
VK_CONTROL
VK_DOWN
VK_ENTER
VK_ESCAPE
VK_LEFT
VK_PAGE_DOWN
VK_PAGE_UP
VK_RIGHT
VK_SHIFT
VK_UP

Класс KeyEvent является подклассом InputEvent.

KeyEvent (Component src, int type, long when, int modifiers, int code, char ch)
src             передается ссылка на компонент, сгенерировавший событие
type            Тип события
when            системное время, когда была нажата клавиша
modifiers       отражает, какие модифицирующие клавиши были нажаты при возникновении события клавиатуры
code            Код виртуальной клавиши,
ch              символьный эквивалент (в случае существования)

Если допустимый символ не существует, то ch будет содержать СНАR_UNDEFINED.
Для событий КЕУ _ TYPED в code указывается VK_UNDEFINED.
_________________________________________________________________________
char getKeyChar ()      возвращает введенный символ
int getKeyCode ()       возвращает введенный код
